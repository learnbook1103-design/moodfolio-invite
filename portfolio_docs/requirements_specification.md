# 📋 요구사항 정의서 (Requirements Specification)
**프로젝트명**: MoodFolio (AI 기반 포트폴리오 생성 서비스)
**작성일**: 2026-01-09

## 1. 개요
본 문서는 MoodFolio 서비스의 관리자 기능 및 시스템 안정성 강화를 위해 구현된 핵심 기능들의 요구사항을 정의합니다.

## 2. 기능 요구사항 (Functional Requirements)

| ID | 카테고리 | 요구사항 명 | 상세 설명 | 우선순위 |
|:---:|:---:|:---|:---|:---:|
| **REQ-ADM-001** | 관리자 | 관리자 로그인 | 관리자 권한이 있는 이메일 계정으로 시스템에 접속할 수 있어야 한다. | High |
| **REQ-ADM-002** | 관리자 | 템플릿 목록 조회 | 시스템에 등록된 12종의 모든 포트폴리오 템플릿 목록을 조회할 수 있어야 한다. | High |
| **REQ-ADM-003** | 관리자 | 템플릿 상태 변경 | 각 템플릿의 활성/비활성 여부를 토글 버튼으로 즉시 제어할 수 있어야 한다. | High |
| **REQ-ADM-004** | 관리자 | AI 사용량 통계 | 기간별(일간/주간/월간) AI 생성 요청 횟수 및 성공/실패 통계를 조회할 수 있어야 한다. | Medium |
| **REQ-SYS-001** | 시스템 | 공개 설정 조회 API (Public) | 인증되지 않은 일반 사용자도 현재 활성화된 템플릿 목록을 조회할 수 있는 API를 제공해야 한다. | High |
| **REQ-SYS-002** | 시스템 | 설정 관리 API (Admin) | 템플릿 상태 변경은 오직 관리자 토큰을 소지한 요청에 대해서만 처리되어야 한다. | High |
| **REQ-USR-001** | 사용자 | 동적 템플릿 선택 | 사용자가 포트폴리오 생성 시, 관리자가 '활성'으로 설정한 템플릿만 선택 화면에 표시되어야 한다. | High |

## 3. 비기능 요구사항 (Non-Functional Requirements)

| ID | 카테고리 | 요구사항 명 | 상세 설명 |
|:---:|:---:|:---|:---|
| **NFR-REL-001** | 신뢰성 | 서버 기동 안정성 | Windows, macOS 등 다양한 OS 환경 및 파일 인코딩(BOM) 차이와 무관하게 백엔드 서버가 100% 정상 기동되어야 한다. |
| **NFR-SEC-001** | 보안 | API 키 보안 | Google API Key 등 민감 정보는 소스코드에 하드코딩되지 않고 환경 변수로만 관리되어야 한다. |
| **NFR-PER-001** | 성능 | UI 반응성 | 관리자 대시보드의 상태 변경은 1초 이내에 완료되어야 하며, 변경 사항은 사용자 페이지에 즉시 반영되어야 한다. |
| **NFR-MNT-001** | 유지보수성 | 협업 환경 | 개발 소스코드(Origin)와 협업용 코드(Invite)의 Git 리모트가 분리되어 관리되어야 한다. |
